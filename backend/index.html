MiraiChat Backend Customization Guide

Language / 言語 / 语言 / 語言: English · 日本語 · 简体中文 · 繁體中文

This guide is for users who can do some hands-on setup (Python, Docker, server config) but are not full-time backend engineers.

If that sounds like you, good news: MiraiChat is intentionally designed so you can customize backend behavior step by step.

Why MiraiChat supports custom backends

MiraiChat is privacy-first, but privacy needs differ by user and by deployment environment.

Some users want complete data ownership.
Some users need region-specific network tuning.
Some users want to integrate chat with their own automation and internal services.

A custom backend lets you keep the same app experience while deciding:

  • where data is stored,
  • how messages are routed,
  • what security and retention policies apply,
  • and which extra features you want to add.

What you can customize in practice

You do not need to rewrite everything. Start from one module at a time.

1) Message routing and delivery strategy

You can tune:

  • online delivery priority,
  • cross-node fanout behavior,
  • offline queue strategy,
  • multi-device self-sync rules.

Result: better control over reliability, latency, and device experience.

2) Authentication and session policy

You can tune:

  • session expiration,
  • registration mode (open or key-based),
  • admin protection rules,
  • token validation strictness.

Result: stronger private deployment security or simpler trusted-network onboarding.

3) WebSocket event pipeline

You can tune:

  • accepted event types,
  • validation logic,
  • event-level routing behavior,
  • handling for receipts/deletes/system signals.

Result: you can add your own event types for custom product workflows.

4) Push notification behavior

You can tune:

  • which message classes trigger push,
  • sync vs async push strategy,
  • provider mode/fallback order,
  • relay behavior for hub/private topologies.

Result: better battery/network balance and privacy-aware notification design.

5) Storage and retention policy

You can tune:

  • media/user storage limits,
  • file retention windows,
  • offline message retention,
  • cleanup scheduler policy.

Result: from low-retention private mode to larger self-hosted archives.

6) Infrastructure path (single-node to multi-node)

You can tune:

  • local SQLite-first operation,
  • in-memory vs Redis presence and fanout,
  • migration path to broader infra.

Result: start simple, then scale only when you actually need it.

Benefits of running your own backend

Custom backend operation gives immediate, practical value:

  • Full ownership: your server, your data, your retention rules.
  • Local control: pick region, storage, and operating policy.
  • Faster iteration: change backend logic and test right away.
  • Integration freedom: connect your own APIs, moderation, identity, or automation.
  • No forced vendor lock-in for model providers or workflow design.

WebSocket modes: Direct vs Tunnel

MiraiChat supports both modes because networking reality differs across users.

Direct WebSocket mode (WS_CONNECTION_MODE=direct)

Best when you have a stable public endpoint.

  • Lower latency in most deployments.
  • Fewer hops and simpler traffic path.
  • Good for predictable performance.

Choose direct when you can expose a reliable inbound endpoint and want maximum speed.

Tunnel WebSocket mode (WS_CONNECTION_MODE=tunnel)

Best when public inbound exposure is difficult.

  • Works well behind strict NAT or home network limits.
  • Easier remote access setup for self-hosted users.
  • Usually adds a bit of latency compared with direct mode.

Choose tunnel when convenience and reachability matter more than absolute lowest latency.

Suggested first customization path

If you are new to backend customization, use this sequence:

  1. Keep default auth/session behavior and verify baseline stability.
  2. Add one small endpoint or one custom WebSocket event.
  3. Tune one retention rule and observe operations for a few days.
  4. Move to Redis fanout only when single-node limits appear.

This incremental approach is safer and easier to maintain.

Final note

You do not need to be a backend specialist to make MiraiChat yours.

Start with one small change, test locally, iterate, and grow your setup at your own pace.
